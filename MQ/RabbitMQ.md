## RabbitMQ 生产者（Producer）可靠性投递

核心流程图解

```
【业务触发】                      【RabbitTemplate】                  【RabbitMQ Broker】
    │                                  │                                   │
    │ 1. 调用 sendMessage()             │                                   │
    ├─────────────────────────────────>│                                   │
    │                                  │ 2. 异步发送消息                    │
    │                                  ├──────────────────────────────────>│
    │ 3. 方法立即返回（非阻塞）           │                                   │
    │<─────────────────────────────────┤                                   │
    │                                  │                                   │ 4. 校验并接收消息
    │                                  │                                   └─┐
    │                                  │                                     │ 
    │                                  │ 5. 异步回传 ACK/NACK <──────────────┘
    │                                  |<──────────────────────────────────
    │                                  │
    │                                  │ 6. 触发 confirmCallback 监听器
    │                                  ├─┐
    │                                  │ │ 7. 根据 ACK 结果更新数据库
    │                                  │ │    (taskInfoService.updateMqSendStatus)
    │                                  │<┘
```



完整的消息生命周期图：

```
[ 生产者 ] 
    │  
    │ 1. 发送消息
    ▼
[ 交换机 (Exchange) ]  ─── 2. 只要消息到这里，Broker 就会立刻
    │                         向生产者触发 ConfirmCallback (ACK)！
    │                         此时，消费者可能还没开机呢。
    ▼
[ 队 列 (Queue) ]
    │
    │ 3. 投递消息
    ▼
[ 消费者 ]  ────────────── 4. 消费成功，向队列发送 Consumer ACK
```
